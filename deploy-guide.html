HWRoom deployment notes

Put HWRoom
on view.

1. Upload your media

Keep two virtual folders in your Blob container: instagram/ and youtube/. Upload the local files into their matching folder. The names in content.json already match your directory listing.

2. Allow browser access

In the storage account’s Resource sharing (CORS) settings for Blob service, add the URL of your Static Web App as an allowed origin. Allow the GET and HEAD methods, the Range request header, and expose Content-Length, Content-Range, and Accept-Ranges. Set a max age such as 86400 seconds.

3. Choose access

The simplest public-gallery setup is anonymous read access on this one media container. If the archive must remain private, place a read-only container SAS in config.json; use a long-lived token only if you accept that anyone who can view the page can also see and reuse it. For stronger private access, a static-only site needs an external proxy or edge authorization layer.

4. Point the catalog at Azure

Open config.json and replace the placeholder with your container URL:

{
  "blobBaseUrl": "https://ACCOUNT.blob.core.windows.net/CONTAINER",
  "sasToken": "",
  "catalogMode": "manifest",
  "includeFolders": ["instagram", "youtube"],
  "videoPreviewFrames": true,
  "autoplayVideos": false
}

manifest mode uses the supplied 44-item content.json. Change catalogMode to blob to discover supported media automatically from Azure. Automatic mode requires container List access (or a SAS containing both Read and List permissions).

When videoPreviewFrames is true, each video lazily loads a tiny range near its first frame to create a thumbnail-like preview. You can still provide a dedicated thumbnail filename in content.json; a real thumbnail is preferred when present.

5. Deploy the page

Deploy this folder as the root of your Azure Static Web App. There is no build step. If using Azure DevOps or GitHub Actions, set the app location to the folder containing index.html and leave the output location empty.

6. Cache the heavy files

On the Blob objects, use a long cache header such as public, max-age=31536000, immutable if file names never change. Keep the JSON catalog on a short cache policy so additions appear quickly. MP4 files should have video/mp4 content type and support byte-range requests.

Adding future work

Upload the new blob, then add an object to content.json. Optional title and thumbnail fields make the wall more descriptive. Video thumbnails can be small WebP or JPEG files stored beside their video.
